# 同步商品功能问题分析文档

## 问题描述

用户反馈：点击"同步商品"后，商品列表的：
1. **可售库存** 又有值了（之前应该为0）
2. **配送渠道** 变成空了

---

## 一、数据流完整梳理

### 1.1 同步商品操作链路

```
用户点击"同步商品"按钮
    ↓
前端 LogisticsDetail.vue: handleSyncProducts()
    ↓
POST /api/logistics/sync-products/:id
    ↓
logistics.controller.js: syncProducts()
    ↓
ProductModel.syncFromLogistics(skuItems, logisticsId, fbaWarehouseNumber, shopId)
    ↓
SQL UPDATE / INSERT amazon_products
```

### 1.2 同步核心代码 (ProductModel.js:355-402)

```javascript
async syncFromLogistics(items, logisticsId, fbaWarehouseNumber, shopId) {
  const batchPrefix = `logistics_${logisticsId}`;

  for (const item of items) {
    const sku = item.sku_code || item.seller_sku || '';
    if (!sku) continue;

    // 按 SKU + 批次匹配
    let existing = await this.findBySkuAndBatch(sku, batchPrefix);

    if (existing) {
      // UPDATE - 只更新这三个字段！
      await this.query(
        `UPDATE amazon_products SET
          item_name = COALESCE(?, item_name),
          shop_id = ?,
          upload_batch = ?
        WHERE id = ?`,
        [item.sku_name || existing.item_name, shopId, batchPrefix, existing.id]
      );
    } else {
      // INSERT - 新建商品，quantity = 0
      await this.create({
        seller_sku: sku,
        item_name: item.sku_name || sku,
        quantity: 0,        // 新建时设为0
        shop_id: shopId,
        upload_batch: batchPrefix
      });
    }
  }
}
```

## 二、问题根因分析

### 2.1 可售库存在同步后又有了值

**原因：** `syncFromLogistics` 的 UPDATE 语句**没有更新 quantity 字段**。

同步逻辑：
- **UPDATE 分支**：只更新 `item_name`, `shop_id`, `upload_batch`，**quantity 保持原值不变**
- **INSERT 分支**：新建时 quantity = 0

如果该 SKU 之前通过其他方式（如 FBA 库存上传）导入了 `amazon_products` 表，并且当时 quantity 有值（比如 100），那么：
1. 同步商品点击后
2. 找到已有记录（existing 不为 null）
3. 执行 UPDATE，**quantity 字段完全没被修改**
4. 查询时 quantity 还是原来的值（100）

### 2.2 配送渠道变成空

**原因：** `syncFromLogistics` 的 UPDATE 语句**没有更新 fulfillment_channel 字段**。

同步逻辑：
- **UPDATE 分支**：只更新 `item_name`, `shop_id`, `upload_batch`，**fulfillment_channel 保持原值不变**
- 如果原记录 fulfillment_channel 是空字符串或 NULL，则同步后仍是空

---

## 三、核心问题

### 问题：同步商品操作的设计意图不清晰

**现状：**
- 物流同步**本意**是：将物流在途的 SKU 同步到商品表，数量固定为 0（因为在途不算可售）
- 但实际代码：**UPDATE 时不修改 quantity**，导致原有数量被保留

**矛盾：**
- INSERT 时 quantity = 0（正确，在途=不可售）
- UPDATE 时 quantity 不变（错误，应该也是 0，或者不更新）

### 问题：fulfillment_channel 字段从未被设置

物流同步流程中，**从未给 fulfillment_channel 赋值**：
1. 物流上传的 SKU 文件没有这个字段
2. syncFromLogistics 也没有写入这个字段
3. 所以要么原值是空的，要么原值被保留

---

## 四、解决方案

### 方案 A：同步时明确设置 quantity = 0（推荐）

```javascript
if (existing) {
  await this.query(
    `UPDATE amazon_products SET
      item_name = COALESCE(?, item_name),
      shop_id = ?,
      upload_batch = ?,
      quantity = 0  -- 明确设为0，物流在途不算可售
    WHERE id = ?`,
    [item.sku_name || existing.item_name, shopId, batchPrefix, existing.id]
  );
}
```

**优点：** 语义清晰，物流同步的商品 quantity 一定是 0
**缺点：** 如果同一个 SKU 有多个物流批次，后一次同步会覆盖前一次的 quantity

### 方案 B：同步时不更新 quantity，让用户手动维护

```javascript
if (existing) {
  await this.query(
    `UPDATE amazon_products SET
      item_name = COALESCE(?, item_name),
      shop_id = ?,
      upload_batch = ?
      -- quantity 不更新，保持原值
    WHERE id = ?`,
    [item.sku_name || existing.item_name, shopId, batchPrefix, existing.id]
  );
}
```

**前提：** 需要确保商品列表的"可售库存"计算方式正确（不应该从 amazon_products.quantity 读取）

### 方案 C：修改商品列表查询逻辑

将"可售库存"改为计算值：
```
可售库存 = amazon_products.quantity + FBA库存 + FBA预留 - FBA预留
```

或者：
```
可售库存 = FBA库存数量（因为物流在途不算可售）
```

## 五、推荐修复步骤

### 立即修复

1. **明确同步行为**：在 syncFromLogistics 的 UPDATE 中添加 `quantity = 0`

2. **或者修改查询逻辑**：
   - 可售库存 = `amazon_products.quantity`（不变）
   - 配送渠道需要从其他途径填充（如 FBA 上传时）

### 长期优化

1. 考虑是否需要 fulfillment_channel 字段
2. 如果需要，确认哪个模块负责写入
3. 考虑添加 sku_inventory 统一库存表（见 plan 文件）
